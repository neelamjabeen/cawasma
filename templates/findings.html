# HTTP Response Headers — /api/v2/users/profile Access-Control-Allow-Origin: * ← CRITICAL: wildcard on auth endpoint Access-Control-Allow-Credentials: true Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS Content-Type: application/json # Combined with credentials=true, this allows any origin to make credentialed requests
Remediation
Replace the wildcard origin with an explicit allowlist of trusted origins. Never combine Access-Control-Allow-Origin: * with Access-Control-Allow-Credentials: true — browsers reject this, but some servers misconfigure it to bypass this restriction.

Fix: Set Access-Control-Allow-Origin: https://yourdomain.com and validate the Origin header server-side against an explicit allowlist.
